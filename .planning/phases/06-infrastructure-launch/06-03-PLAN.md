  <action>
    Create OWNER.md in the repository root. Write in plain language for a non-developer audience. No jargon. Use GitHub web editor instructions (not command line).

    The guide must cover these sections:

    **1. Adding a News Article**
    - Go to the `content/news/` folder on GitHub
    - Click "Add file" -> "Create new file"
    - Name the file with a slug (e.g., `2026-season-opener.md`)
    - Provide the exact frontmatter template:
      ```
      ---
      title: "Your Article Title"
      date: "YYYY-MM-DD"
      excerpt: "One sentence summary."
      image: "/images/news/optional-image.jpg"
      tag: "Announcement"
      ---

      Article body here. Use plain text or basic markdown.
      ```
    - Explain: commit the file, Vercel auto-deploys

    **2. Changing the Google Sheet**
    - Go to Vercel Dashboard -> Project -> Settings -> Environment Variables
    - Find SPREADSHEET_ID, click edit, paste new value
    - Redeploy: Deployments tab -> three-dot menu on latest -> Redeploy

    **3. Adding Team Logos**
    - Upload logo image to `public/images/teams/` via GitHub web editor
    - Name must match the team's `logoPath` value in `src/config/teams.ts`
    - Explain the naming convention (e.g., `/images/teams/thunder.png`)

    **4. Adding Player/News Images**
    - Upload to `public/images/players/` or `public/images/news/` via GitHub
    - Reference in news frontmatter as `/images/news/filename.jpg`

    **5. Triggering a Redeploy**
    - Any push to main auto-deploys
    - Manual: Vercel Dashboard -> Deployments -> Redeploy

    **6. Troubleshooting**
    - "All pages show errors" -> Check SPREADSHEET_ID is set in Vercel
    - "New article not showing" -> Check frontmatter format, especially date format
    - "Build failed" -> Check Vercel deployment logs for error message

    Keep the entire document under 150 lines. Use short sentences. Include the exact URLs where possible (e.g., "Go to vercel.com/dashboard").
