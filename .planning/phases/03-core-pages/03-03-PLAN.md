**Data fetching (add to existing):**

1. `/api/stats?category=hitting&year=${selectedYear}` — filter client-side by team name.
2. `/api/stats?category=pitching&year=${selectedYear}` — filter client-side by team name.

Refetch when `year` changes. Match players to this team by comparing the `team` field to `team.name` or `team.uniqueTeamName` (case-insensitive).

**Add PageNavigation:**
Import and render `PageNavigation` at the top (currently missing).

**Roster section:**
1. Add a "Roster" section below the season summary.
2. Display players as cards in a responsive grid (`grid-cols-2 md:grid-cols-3 lg:grid-cols-4`).
3. Each player card shows:
   - Player name (from hitting data, or pitching if player only pitches).
   - Position indicator: "Hitter" / "Pitcher" / "Both" based on which datasets they appear in.
   - Key stat: for hitters show AVG and HR, for pitchers show ERA and K.
   - The card is a Link to `/stats/players/${player.playerId}` (the player detail page).
4. Deduplicate players who appear in both hitting and pitching — show them once with "Both" designation.
5. Card styling: `bg-white rounded-lg shadow-sm border p-4 hover:shadow-md transition-shadow`.
6. If no players found for the selected year/team, show "No roster data available for this season."

**Team aggregate stats section:**
1. Add a "Team Stats" section below the roster.
2. Two subsections: "Team Batting" and "Team Pitching".
3. Team Batting aggregate: calculate from the team's hitters for the selected year:
   - Games (max of player games), AVG (team total H / total AB), HR (sum), RBI (sum), OPS (team avg), wRC+ (team avg of qualified hitters).
   - Display as a stat card grid similar to the existing season summary.
4. Team Pitching aggregate: calculate from the team's pitchers:
   - ERA (team total ER / total IP * 9), WHIP (team (H+BB) / IP), K (sum), W-L (sum wins, sum losses).
   - Display as a stat card grid.
5. Below each aggregate, show individual player breakdown in a compact table:
   - Batting table: Player, G, AVG, HR, RBI, OPS — sorted by PA descending.
   - Pitching table: Player, G, IP, ERA, K, W-L — sorted by IP descending.
   - Player names are Links to their detail pages.
