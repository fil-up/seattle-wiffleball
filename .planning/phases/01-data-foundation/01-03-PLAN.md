Migrate all client-side components from fetching Google Sheets directly to fetching from the internal API routes built in Plan 01-02. Purpose: DATA-04 (proxy all Sheets data through API routes) — the final step in establishing the single data pipeline. No more client-side Google Sheets fetches means better caching, no CORS concerns, and a single place to debug data issues.
Output: All data-displaying components fetch from /api/* routes; zero direct Google Sheets URLs in client code.
</objective>
