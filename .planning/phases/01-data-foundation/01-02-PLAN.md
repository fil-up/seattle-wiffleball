  <name>Task 1: Build the shared Google Sheets utility (src/lib/sheets.ts)</name>
  <files>src/lib/sheets.ts</files>
  <action>
    Create `src/lib/sheets.ts` — the single module that owns ALL Google Sheets data access. This module must handle:

    **1. Environment configuration**
    Read `SPREADSHEET_ID` from `process.env.SPREADSHEET_ID`. Throw a clear error at module load if the variable is missing.

    **2. gviz URL construction**
    `buildGvizUrl(sheet: string, range: string): string`
    - URL format: `https://docs.google.com/spreadsheets/d/${SPREADSHEET_ID}/gviz/tq?tqx=out:json&sheet=${encodeURIComponent(sheet)}&range=${range}`
    - Sheet name must be URI-encoded (e.g., "Player/Team Adj" → "Player%2FTeam%20Adj")

    **3. gviz response parsing**
    `parseGvizResponse(text: string): GvizTable`
    - Use regex to extract JSON from the JSONP wrapper: `text.match(/google\.visualization\.Query\.setResponse\(({.*})\);?\s*$/s)`
    - DO NOT use `substring(47).slice(0, -2)` — that magic number is fragile and breaks when Google changes the response prefix
    - Define a `GvizTable` interface for the parsed response structure:
      ```
      interface GvizTable {
        table: {
          cols: Array<{ id: string; label: string; type: string }>
          rows: Array<{ c: Array<{ v: any; f?: string } | null> }>
        }
      }
      ```
    - The `cols` array preserves header labels — this is important for Phase 2 header-based column mapping (STAT-03). Don't discard it.

    **4. In-memory cache with TTL and stale-while-revalidate**
    - Use a module-level `Map<string, CacheEntry>` for cache storage
    - Cache key format: `${sheet}!${range}` — this correctly distinguishes split ranges within the same tab (e.g., IH aggregates vs IH yearly totals)
    - TTL: 30 seconds (`CACHE_TTL_MS = 30_000`)
    - On cache hit within TTL: return cached data immediately, `stale: false`
    - On cache miss or expired: fetch from Google Sheets
    - On fetch failure with expired cache: return stale cached data with `stale: true`
    - On fetch failure with no cache: throw error (first-load failure)
    - CacheEntry shape: `{ data: T; fetchedAt: number }`

    **5. Main export: fetchSheet**
    ```typescript
    export async function fetchSheet<T>(
      sheet: string,
      range: string,
      transform: (rows: GvizRow[], cols: GvizCol[]) => T
    ): Promise<{ data: T; stale: boolean }>
    ```
    - Generic over the transform output type
    - Accepts a `transform` function that converts raw gviz rows+cols into the desired shape
    - Returns `{ data, stale }` — every consumer gets staleness information

    **6. Cell extraction helpers**
    Export these helpers for use in transform functions:
    - `cellValue(row, index)` — raw value or null
    - `cellString(row, index)` — string coercion with fallback to ''
    - `cellNumber(row, index)` — numeric coercion with fallback to 0
    - `cellFloat(row, index, decimals?)` — float with optional rounding

    These helpers handle the gviz cell format where each cell is `{ v: value, f?: formattedValue } | null`.
